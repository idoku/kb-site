Checklist: Document Quality

[ ] 标题能直接说明文档对象。 [ ] 背景说明了真实问题，而不是只描述想法。 [ ] 目标和非目标分开。 [ ] 关键名词首次出现时有解释或上下文。

Clarity

  • [ ] 标题能直接说明文档对象。
  • [ ] 背景说明了真实问题，而不是只描述想法。
  • [ ] 目标和非目标分开。
  • [ ] 关键名词首次出现时有解释或上下文。

Boundary

  • [ ] 写清楚 owner。
  • [ ] 写清楚长期资产放哪里。
  • [ ] 写清楚临时产物放哪里。
  • [ ] 写清楚哪些内容不应回传或发布。

Executability

  • [ ] 有明确执行步骤。
  • [ ] 有可验证验收标准。
  • [ ] 有必要的命令、路径或接口示例。
  • [ ] 没有依赖读者猜测的隐含前提。

Maintainability

  • [ ] 不把工具实现和知识资产混在一起。
  • [ ] 不把一次性项目文档误当成通用模板。
  • [ ] 样例已脱敏。
  • [ ] 后续工作没有阻塞本轮落地。
